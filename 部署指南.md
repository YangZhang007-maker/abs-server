# ABS 存续期智能提醒系统 — 云端部署指南

## 架构

```
用户浏览器 → Vercel (前端) → Render (后端 API) → Supabase (数据库 + 文件存储)
```

---

## 一、Supabase 配置（⭐ 必须先做）

### 1.1 获取数据库连接信息

进入 Supabase 控制台 → Settings → Database：

| 设置项 | 值 |
|--------|-----|
| Host | `aws-0-us-west-1.pooler.supabase.com` |
| Port | `6543` |
| User | `postgres.buklmnpbbkfrkdvmvupr` |
| Database | `postgres` |
| Password | 你的 Supabase 数据库密码 |

### 1.2 获取 Service Role Key

Supabase 控制台 → Settings → API → **service_role** key（以 `eyJ` 开头的一长串）

> ⚠️ 这是敏感密钥，不要提交到 GitHub

### 1.3 配置 Storage Bucket 权限

1. 进入 Supabase → Storage → `abs-documents` bucket
2. 点击 **Policies** → **New Policy**
3. 选择 `For full customization`，粘贴以下 SQL：

```sql
-- 允许认证用户上传文件
CREATE POLICY "允许认证用户上传" ON storage.objects
  FOR INSERT WITH CHECK (bucket_id = 'abs-documents');

-- 允许任何人下载文件（通过签名 URL）
CREATE POLICY "允许公开下载" ON storage.objects
  FOR SELECT USING (bucket_id = 'abs-documents');

-- 允许认证用户删除文件
CREATE POLICY "允许认证用户删除" ON storage.objects
  FOR DELETE USING (bucket_id = 'abs-documents');
```

## 二、Render 部署后端

### 2.1 推送代码到 GitHub

```bash
cd ABS-server
git init
git add .
git commit -m "Deploy to Render"
git remote add origin https://github.com/你的用户名/abs-server.git
git push -u origin main
```

### 2.2 在 Render 创建 Web Service

1. 登录 [render.com](https://render.com)
2. 点击 **New +** → **Web Service**
3. 连接你的 GitHub 仓库
4. 配置：

| 设置项 | 值 |
|--------|-----|
| Name | `abs-backend` |
| Root Directory | `backend` |
| Build Command | `npm install --legacy-peer-deps && npm run build` |
| Start Command | `node dist/main.js` |

### 2.3 添加环境变量

在 Render 的 Environment 标签页中添加：

| KEY | VALUE |
|-----|-------|
| `DB_HOST` | `aws-0-us-west-1.pooler.supabase.com` |
| `DB_PORT` | `6543` |
| `DB_USERNAME` | `postgres.buklmnpbbkfrkdvmvupr` |
| `DB_PASSWORD` | **你的 Supabase 数据库密码** |
| `DB_DATABASE` | `postgres` |
| `JWT_SECRET` | `abs-jwt-production-secret-2026-please-change-me` |
| `SUPABASE_URL` | `https://buklmnpbbkfrkdvmvupr.supabase.co` |
| `SUPABASE_SERVICE_KEY` | **你的 Supabase Service Role Key** |
| `SUPABASE_STORAGE_BUCKET` | `abs-documents` |

### 2.4 部署

点击 **Create Web Service**，等待 3-5 分钟。

部署成功后，记下 Render 给你的域名（如 `abs-backend.onrender.com`）。

## 三、Vercel 部署前端

### 3.1 更新前端 API 地址

修改 `frontend/src/api/client.ts` 第 5 行，把 `你的Render后端地址` 替换为实际域名：

```typescript
const BASE_URL = import.meta.env.PROD
  ? 'https://abs-backend.onrender.com/api/v1'   // ← 改成你的 Render 域名
  : '/api/v1';
```

### 3.2 部署到 Vercel

1. 登录 [vercel.com](https://vercel.com)
2. 点击 **Add New** → **Project**
3. 导入你的 GitHub 仓库
4. 配置：

| 设置项 | 值 |
|--------|-----|
| Framework Preset | Vite |
| Root Directory | `frontend` |
| Build Command | `npm install --legacy-peer-deps && npm run build` |
| Output Directory | `dist` |

5. 点击 **Deploy**

部署成功后，Vercel 会给你一个域名（如 `abs-frontend.vercel.app`）。

## 常见问题

### Q: Render 部署失败？
确保 `backend/` 下有 `dist/` 目录（编译后的代码）。如果没有，本地先运行 `npm run build`。

### Q: 上传文件 404？
检查 Supabase Storage Bucket 的 RLS 策略是否已正确配置。

### Q: 前端访问后端跨域？
后端 `main.ts` 已配置 `app.enableCors({ origin: true })`，允许所有来源。

### Q: 数据库连接失败？
检查 Supabase 数据库是否在运行（Settings → Database → 状态）。如果使用 Supabase 免费版，数据库在 1 周不活动后会暂停，需手动恢复。
